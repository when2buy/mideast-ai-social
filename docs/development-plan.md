# 开发计划

## 项目背景

开发一款面向中东市场的AI虚拟角色社交平台，对标 Talkie 和 Linky AI。

## 待确认：产品方向

> ⚠️ **需要 Penny 确认**：直播 app 和 AI 虚拟主播是两个完全不同的方向，需要先确定做哪个。

| | 方向A：AI虚拟角色聊天 | 方向B：真人直播/语音房 |
|---|---|---|
| **对标** | Talkie, Linky AI, Character.AI | Yalla, Bigo Live, Hakuna |
| **核心体验** | 用户和AI角色文字/语音聊天 | 真人主播直播、多人语音房 |
| **技术重点** | LLM、TTS、角色系统 | 音视频SDK、实时通信、CDN |
| **变现** | 订阅制、角色解锁 | 礼物打赏、会员 |
| **开发难度** | 中等（主要是AI集成） | 较高（音视频基础设施复杂） |
| **Demo周期** | 1-2天可出原型 | 3-5天（依赖第三方SDK） |

---

## 开发原则（Steve确认）

1. **先做轻量级Demo** — 1-2天搞定，验证可行性
2. **第一版只做iOS** — 最小化范围
3. **用现成SDK** — 不从头造轮子
4. **云平台用DigitalOcean** — 利用现有基础设施
5. **先看效果再决定下一步** — MVP思维

---

## 方向A：AI虚拟角色聊天（如确认）

### Phase 0 — Demo（1-2天）

**目标**：可运行的iOS原型，能和AI角色文字+语音聊天

**功能**：
- 3-5个预设AI角色（阿拉伯语）
- 文字聊天（接Gemini API）
- AI语音回复（接TTS）
- 简单UI，支持RTL布局

**技术栈**：
- 前端：React Native（或Flutter，待定）
- AI：Google Gemini 2.5 API
- TTS：ElevenLabs Turbo v3（阿拉伯语）
- 后端：Node.js + Express（轻量API层）
- 部署：DigitalOcean

### Phase 1 — MVP（4-6周）

在Demo基础上补齐：
- [ ] 用户注册/登录（手机号+OTP）
- [ ] 角色浏览页面（卡片式）
- [ ] 聊天记录持久化
- [ ] 语音输入（Whisper ASR）
- [ ] 基础角色创建功能
- [ ] 阿拉伯语本地化完善

### Phase 2 — 完善（4-6周）

- [ ] 角色卡片收集系统
- [ ] 订阅/付费系统（Apple IAP）
- [ ] 社区功能（分享角色）
- [ ] 多语言支持（英语、波斯语）
- [ ] Android版本
- [ ] 内容审核系统

## 方向B：真人直播/语音房（如确认）

### Phase 0 — Demo（2-3天）

**目标**：可运行的iOS原型，支持多人语音房

**功能**：
- 创建/加入语音房间
- 多人实时语音（最多8人）
- 简单的房间列表UI
- 基础用户身份

**技术栈**：
- 前端：React Native
- 音视频：Agora / ZEGO SDK
- 后端：Node.js + Socket.io
- 部署：DigitalOcean

### Phase 1 — MVP（6-8周）

- [ ] 用户系统
- [ ] 房间管理（上麦/下麦/踢人）
- [ ] 礼物/打赏系统
- [ ] 主播个人页
- [ ] 房间分类/推荐
- [ ] 基础内容审核
